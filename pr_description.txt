Исправляет ложное срабатывание проверки `repeated_followup_mentions` в скрипте ревью качества автономных PR для задач типа `automation-health`.

Из-за того, что acceptance criteria таких задач часто содержат фразу "A follow-up Automation Health run no longer reports...", скрипт ошибочно блокировал PR с ошибкой quality_failure (PR body repeatedly mentions follow-up tasks).

Это исправление обобщает исключение с `automation-health-repeated-followup` на все задачи, начинающиеся с `automation-health-`, что позволяет таким PR проходить quality gate.
